Methodology and curation workflow

Project XC is built from layered evidence. The imported Libxc layer gives broad discovery coverage. Curated Project XC records add interpretation, aliases, parameters, and caveats.

Validation gates

  1. Import gate: Libxc parser must produce a large, duplicate-free snapshot with DOI-shaped references.
  2. Schema gate: curated records must include names, aliases, rung, kind, ingredients, components, references, notes, and status.
  3. Reference gate: every scientific claim should be linked to DOI, program documentation, source code, or a reproducible input/output test.
  4. Website gate: the site must build from data only; no hand-edited catalog HTML.
  5. Agent-review gate: importer, curation, alias, and QA agents review each other before records become verified.

Known ambiguity classes

  • B3LYP VWN3/VWN5/RPA convention differences.
  • Range-separated functionals where omega and exact-exchange fractions are input-dependent.
  • Dispersion-corrected names where the correction may be outside the XC library.
  • Nonlocal correlation terms such as VV10 that may require separate grids/kernels.
  • Meta-GGA grid sensitivity and kinetic-energy-density conventions.
